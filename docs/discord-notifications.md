# Discord notification sources

Inventory of every component in `/opt/docker` that posts to Discord, grouped by webhook. Written 2026-04-24 during the rotation audit.

Three distinct Discord webhooks are in active use. The bulk of the system posts through the first one via a shared helper; the other two are isolated per-feature channels owned by gluetun side-scripts.

---

## Webhook 1: Ops channel

**Webhook ID:** `1497253937403789513` (rotated 2026-04-24; previous `1481247184874897439` leaked in git history — revoked in Discord)

**Storage (two files, same value — duplicated by design):**
- `apps/watchtower/.env` → `AUTO_UPDATE_NOTIFICATION_URL=discord://<token>@<id>` (shoutrrr form — consumed directly by Watchtower and parsed by `notify-discord.sh`).
- `apps/singbox-profiles/.secrets.yaml` → `users.<admin>.notify_webhook` (REST form — baked into each admin's laptop installer at render time so it doesn't depend on reading another app's env file at runtime).

**Canonical source:** the watchtower `.env`. `notify-discord.sh` (`/opt/docker/scripts/notify-discord.sh`) reads this file, parses the shoutrrr `discord://token@id` form, and exposes:
- `discord_notify "msg"` — jq-encoded POST, multi-line / special-char safe, best-effort no-op on failure.
- `_discord_webhook_url` — rebuilt REST URL for callers that need raw curl (e.g. multi-line embed payloads).

### Indirect consumers (Windows-laptop installers)

Admin users' laptop installers (generated by `singbox-profiles/generate-installer.sh` from `install-singbox.template.ps1`) receive the REST URL baked into their PowerShell via the `__WEBHOOK_URL__` placeholder, sourced from `users.<admin>.notify_webhook` in `.secrets.yaml`.

The installer posts on: installer completion, config replacement, binary upgrade, config-check failure, binary smoke-test failure, schema mismatch, service-failed-to-reach-Running, and updater exceptions. Each message is prefixed with `[laptop <COMPUTERNAME>]` so multi-admin setups are distinguishable.

**Which users are admin:** determined by the `admin: true` flag in `profiles.yaml`; non-admin users (Carol, Olivier) get an empty `$WEBHOOK_URL` in their installer and emit no notifications.

### Rotation impact

Rotating this webhook requires updating **two files** in lockstep:
1. `apps/watchtower/.env` → `AUTO_UPDATE_NOTIFICATION_URL` (shoutrrr form)
2. `apps/singbox-profiles/.secrets.yaml` → `users.*.notify_webhook` for every admin (REST form)

After editing `.secrets.yaml`, run `./render.py -y` in `apps/singbox-profiles/` to regenerate each admin's installer with the new URL baked in. Admins then re-download their installer (or the next auto-sync pulls it). Watchtower needs a `docker compose up -d watchtower` to pick up the env change.

## Webhook 2: Gluetun home-IP channel (shared: IP-check + gost-check)

**Webhook ID:** `1497254904308301905` (rotated 2026-04-24; previous two IDs `1489316256481021994` + `1489323124913868990` leaked in git history — revoked in Discord and consolidated onto one channel).

**Storage:** `apps/gluetun/webhook.env` → both `IP_CHECK_WEBHOOK_URL` and `GOST_CHECK_WEBHOOK_URL` point at the same URL. Kept as two env vars so the split can be restored later without touching scripts.

**Consumers:**
- `apps/gluetun/check-home-ip.sh` — cron `*/5 * * * *`; posts when home IP drifts off its pinned exit, matches Oracle's egress (leak), or the script errors.
- `apps/gluetun/check-gost.sh` — cron `*/5 * * * *`; posts when gost forwarding probes fail.

**Historical note (2026-04-24):** these used to be two separate channels (webhook 2: IP-check, webhook 3: gost-check) so gluetun's 5-min alert volume couldn't drown ops. On rotation they were consolidated — gluetun outages tend to be short and the two signals overlap in stakeholder. If gluetun ever becomes chattier (e.g. an hours-long tunnel outage generating 12+ messages/hour), split back out by generating a second webhook and pointing `GOST_CHECK_WEBHOOK_URL` at it.

### Rotation impact

Edit `apps/gluetun/webhook.env`. No service restart — both scripts source the file fresh on each 5-minute tick.
